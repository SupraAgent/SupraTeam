## 4.2 Mail-0/Zero Technical Deep Dive

| Layer | Technology |
|-------|-----------|
| Frontend | React 19 + React Router + Vite (NOT Next.js) |
| Backend | Hono + tRPC |
| Database | PostgreSQL via Drizzle ORM |
| Auth | Better Auth |
| AI | Vercel AI SDK, multi-provider (OpenAI, Anthropic, Gemini, Groq) |
| Deployment | Cloudflare Workers + Durable Objects + KV + R2 |

**Key patterns reusable in SupraCRM:**
- `MailDriver` interface (provider-agnostic abstraction)
- Writing style matrix for AI personalization
- Gmail API driver with Pub/Sub for real-time
- TipTap for rich text composition
- No email content stored in DB

**What doesn't port:** Cloudflare infra, Better Auth, `effect` library pipeline.

## 4.3 Email Integration Plan Summary

# Part 6: Improvement Methodology

## 6.1 The 5-Step Loop

```
1. TEAM        → 5 AI personas (roles + expertise)
2. APP         → Define what you're building
3. BENCHMARK   → 3 reference apps, scored 1-100
4. SELF-SCORE  → Rate your app on same criteria
5. IMPROVE     → Team makes changes → rescore → repeat until gap < 10
```

**Rules per round:**
- ONE change per round (isolate impact)
- Score can't go down (revert if it does)
- No persona overruled 3 rounds in a row
- Full team retro every 5 rounds
- Max 20 rounds per session

**Consensus:** Weighted voting (Product 1.2x, Eng 1.0x, Design 1.0x, Growth 0.8x, QA 0.8x). 67%+ = pass.
